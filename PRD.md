- **버전**: v1.0
- **작성일**: 2026-05-22
- **원천 자료**: `26.Q2 사업전략팀 OKR 트래킹` 컨플루언스 페이지 (PDF 첨부)

## 한 줄 요약

컨플루언스 표로 운영하던 사업전략팀 OKR 트래킹을 **Vercel 기반 시각 대시보드 + PDF 보고서**로 전환해, 매주 갱신과 상사 보고를 한 화면에서 해결한다.

---

## 용어 정의

이 문서 전체에서 사용하는 용어. 먼저 읽고 본문으로 이동할 것.

| 용어 | 정의 |
|---|---|
| **Objective (O)** | 분기 단위 상위 목표. 예: "O1. 신규 수주를 확보한다" |
| **Key Result (KR)** | Objective를 달성하기 위한 측정 가능한 핵심 결과. 한 Objective에 여러 KR이 속함 |
| **Commit OKR** | 반드시 달성해야 하는 유형. **진척도 1.0**이 기본 기대치 |
| **Aspire OKR** | 도전적 유형. **진척도 0.6~0.7**이 성공 |
| **진척도(Progress)** | KR의 달성률. 0.0 ~ 1.0+ 사이 실수. **사용자가 직접 % 단위로 수동 입력** |
| **신뢰도(Confidence)** | Owner가 판단하는 달성 가능성. **3단계: 高 / 中 / 低** |
| **달성 완료** | Commit은 진척도 ≥1.0, Aspire는 진척도 ≥0.6 도달 상태 |
| **분기(Quarter)** | 3개월 단위 운영 사이클. 예: 26.Q2 = 2026.04.01 ~ 06.30 |
| **Owner** | KR 담당자. 한 KR에 복수 지정 가능 (예: @benjamin @ria @que) |
| **KPT 회고** | 분기 종료 시 작성하는 회고. Keep(잘된 것) / Problem(문제) / Try(다음에 시도) 세 항목 |

## 1. 배경

사내 OKR 가이드는 Commit과 Aspire 두 유형으로 운영된다. 사업전략팀은 분기마다 OKR을 설정하고 **매주 금요일** 진척도와 신뢰도를 갱신하며, 분기 종료 직전(약 종료 1주 전)에는 KR별 최종 점수표와 멤버별 KPT 회고를 작성한다.

현재 이 운영은 컨플루언스 표 형식으로 이루어지고 있으나, 시각화·편집성·이력 관리·보고 산출물 측면에서 한계가 있어 26.Q2부터 별도 대시보드로 전환한다.

## 2. 문제 정의

- 컨플루언스 표는 진척도·신뢰도·달성 여부가 한눈에 들어오지 않아 상사가 위험 KR이나 분기 경과율을 빠르게 파악하기 어렵다
- 매주 표 셀에 직접 타이핑하는 방식은 편집자에게 부담이 된다
- 분기가 바뀔 때마다 새 페이지를 처음부터 만들어 분기 간 이력이 단절된다
- 보고용 산출물(상사 전달용)이 별도로 가공되어야 한다

## 3. 대상 사용자

| 구분 | 대상 | 주요 행위 |
|---|---|---|
| 편집자 | 사업전략팀 멤버 (구성은 분기마다 가변) | 매주 KR 갱신, 분기 전환 운영, KPT 작성 |
| 조회자 | 상사 (보고 수신자) | 대시보드 조회 또는 PDF 보고서 확인 |

> 권한은 역할로 구분하지 않는다. 링크 보유자 전원이 보기·편집 권한을 갖는다.

## 4. 목표 및 성공 지표

### 제품 목표
- 상사가 **10초 이내**에 분기 진행률·위험 KR·전체 신뢰도를 파악할 수 있다
- 편집자가 **3분 이내**에 매주 갱신을 완료할 수 있다
- 분기 종료 시 별도 가공 없이 **PDF 한 번 다운로드로 보고**가 완료된다

### 성공 지표
| 지표 | 측정 방법 |
|---|---|
| 이번 주 갱신율 | 헤더에 "이번 주 갱신 N/M"으로 항상 노출. 매주 100% 목표 |
| 보고서 활용성 | 분기 종료 후 7일 이내 PDF만으로 상사 보고 완료 (별도 가공 없음) |

## 5. 포함 범위

- 분기 메타 헤더 (분기명, 기간, 경과율, 남은 주차, 이번 주 갱신율)
- Hero 요약 카드 4종 (전체 평균 진척도, 달성 KR 수, 평균 신뢰도, 분기 경과율)
- Objective → KR 계층 구조
- KR 카드 (유형, Owner, 목표, 현재값, 진척도, 신뢰도, 갱신일)
- 인라인 편집
- 분기 셀렉터 (드롭다운)
- 새 분기 생성 (빈 폼)
- 멤버 관리 (동적 추가·삭제)
- 분기 점수표 (KR별 최종 결과·평가)
- KPT 회고 (멤버별 Keep/Problem/Try)
- PDF 다운로드 (분기 보고서)

## 7. 사용자 흐름

### 7-1. 조회자 (상사)
1. URL 접속
2. 분기 셀렉터에서 분기 선택 (기본값: 현재 분기)
3. Hero 카드 → KR 카드 순으로 스크롤하며 현황 파악
4. 필요 시 "PDF 다운로드" 버튼으로 보고서 저장

### 7-2. 편집자 — 매주 금요일 갱신
1. URL 접속, 본인 담당 KR 카드 클릭
2. 현재값(숫자 + 상세 메모), 진척도(%), 신뢰도(高/中/低) 입력
3. 저장 → Hero 카드와 카드 색상 즉시 반영, 갱신일 자동 기록

### 7-3. 편집자 — 분기 종료 회고
1. 점수표·KPT 섹션은 항상 페이지 하단에 노출되어 있음
2. KR별 결과·평가 입력
3. 멤버별 Keep/Problem/Try 입력
4. PDF 다운로드로 최종 보고서 생성

### 7-4. 편집자 — 새 분기 시작
1. "새 분기 만들기" 클릭
2. 분기명·시작일·종료일 입력
3. 빈 폼이 생성되며 활성화. 이전 분기는 아카이브 상태로 전환 (드롭다운에서 계속 조회 가능)

### 7-5. 편집자 — 멤버 관리
1. "멤버 관리" 메뉴 진입
2. 멤버 이름 추가 또는 삭제
3. KR Owner 선택 드롭다운에 즉시 반영

## 9. 지표/데이터 정의

### 9-1. 저장소
- **Vercel KV** (키-값 저장소)

### 9-2. 키 스키마
```
quarters:list                       → [분기 ID 배열]
quarter:{qid}                       → { id, name, start_date, end_date, status: "active" | "archived" }
quarter:{qid}:objectives            → [Objective ID 배열]
quarter:{qid}:objective:{oid}       → { id, title }
quarter:{qid}:objective:{oid}:krs   → [KR ID 배열]
quarter:{qid}:kr:{kid}              → {
                                        id,
                                        type: "Commit" | "Aspire",
                                        owners: [멤버 ID 배열],
                                        target_text,
                                        current_value,        // 숫자
                                        current_detail,       // 상세 메모 텍스트
                                        progress,             // 0.0 ~ 1.0+ 실수
                                        confidence,           // "high" | "mid" | "low"
                                        updated_at            // ISO 날짜
                                      }
quarter:{qid}:final_score:{kid}     → { result_text, evaluation }
quarter:{qid}:retro:{member_id}     → { keep, problem, try }
members:list                        → [멤버 명단]
```

### 9-3. 계산 공식

| 지표 | 계산 |
|---|---|
| 분기 경과율 | `(오늘 - 시작일) / (종료일 - 시작일)` × 100, 0~100% |
| 남은 주차 | `ceil((종료일 - 오늘) / 7)` |
| 현재 주차 | `ceil((오늘 - 시작일) / 7)` |
| 이번 주 갱신율 | `updated_at`이 **이번 주 월요일 0시 이후**인 KR 수 / 전체 KR 수. 매주 월요일 0시에 카운트가 리셋된다 |
| KR 달성 여부 | `(type=Commit AND progress≥1.0) OR (type=Aspire AND progress≥0.6)` |
| 전체 평균 진척도 | 현재 분기 전체 KR `progress`의 산술 평균 |
| Objective 평균 진척도 | 산하 KR `progress`의 산술 평균 |
| 평균 신뢰도 | 高=3, 中=2, 低=1로 환산해 산술 평균 후, 반올림하여 단계로 환원. 2.5 이상 → 高, 1.5 이상 ~ 2.5 미만 → 中, 1.5 미만 → 低 |

### 9-4. 색상 임계값 (전역 고정)

진척도와 신뢰도에 대해 모든 시각 요소가 이 색상을 따른다.

**진척도 바 / Hero "전체 평균 진척도" 카드**
| 진척도 | 색상 |
|---|---|
| ≥60% | 🟢 초록 |
| 30% ~ 60% 미만 | 🟡 노랑 |
| <30% | 🔴 빨강 |
| 0%이고 아직 갱신 없음 | ⚪ 회색 |

> Commit 유형의 KR이 60~99%이면 색은 노랑/초록이되, "달성 완료" 라벨은 1.0 도달 시에만 표시한다.

**신뢰도 신호등**
| 단계 | 아이콘 |
|---|---|
| 高 | 🟢 |
| 中 | 🟡 |
| 低 | 🔴 |

**위험 KR 배지**
- 진척도 <30% **또는** 신뢰도 低 → 카드 좌측 빨간 띠 + 🔴 "위험" 배지

**달성 KR 배지**
- 달성 완료 조건 충족 → 카드 좌측 초록 띠 + ✓ + "달성 완료" 라벨

**배지 우선순위 (한 KR에 동시 적용 시)**
- 달성 완료 > 위험. 달성 조건을 충족한 KR은 신뢰도가 低여도 위험 배지를 표시하지 않는다 (이미 달성했으므로)

## 11. 수용 기준

Given / When / Then 형식. 각 AC는 독립적으로 검증 가능.

**AC-01 (F-01 분기 메타)**
- Given 분기 시작일 2026-04-01, 종료일 2026-06-30이 설정된 상태에서
- When 사용자가 2026-05-22에 대시보드에 접속하면
- Then 헤더에 "경과율 약 57%", "남은 기간 약 6주"가 표시된다

**AC-02 (F-02 Hero 카드)**
- Given 분기 데이터가 입력된 상태에서
- When 사용자가 페이지를 열면
- Then 최상단에 ① 전체 평균 진척도(%) ② 달성 KR 수(N/M) ③ 평균 신뢰도(高/中/低) ④ 분기 경과율(%) 4종 카드가 큰 숫자 + 상태 색상으로 표시된다

**AC-03 (F-04 인라인 편집)**
- Given 편집자가 KR 카드를 보고 있는 상태에서
- When 카드를 클릭하고 현재값, 진척도, 신뢰도를 입력 후 저장하면
- Then Hero 카드와 KR 카드 색상이 즉시 갱신되고 갱신일이 오늘 날짜로 자동 기록된다

**AC-04 (F-05 Aspire 달성)**
- Given Aspire 유형 KR이 있는 상태에서
- When 진척도가 0.6 이상으로 저장되면
- Then 해당 KR 카드 좌측에 초록 띠 + ✓ + "달성 완료" 라벨이 표시된다

**AC-05 (F-05 Commit 달성)**
- Given Commit 유형 KR이 있는 상태에서
- When 진척도가 1.0 미만이면
- Then "달성 완료" 라벨은 표시되지 않는다

**AC-06 (F-05 달성 후 갱신)**
- Given Aspire 유형 KR의 진척도가 0.7로 저장되어 "달성 완료" 라벨이 표시된 상태에서
- When 편집자가 진척도를 0.85로 다시 저장하면
- Then 라벨은 유지된 채 진척도 값과 Hero 카드가 갱신된다

**AC-07 (F-08 위험 KR)**
- Given 진척도 0.2, 신뢰도 低인 KR이 있는 상태에서
- When 사용자가 대시보드를 보면
- Then 해당 KR 카드 좌측에 빨간 띠와 🔴 "위험" 배지가 표시된다

**AC-08 (F-10 과거 분기 읽기 전용)**
- Given 26.Q1이 아카이브 상태인 상태에서
- When 사용자가 분기 셀렉터에서 26.Q1을 선택하면
- Then 모든 편집 UI(인라인 편집, 삭제, 추가, 멤버 관리)가 비활성화된다

**AC-09 (F-11 새 분기 생성)**
- Given 26.Q2가 활성 상태인 상태에서
- When 사용자가 "새 분기 만들기"로 26.Q3를 생성하면
- Then 26.Q3가 빈 폼으로 활성화되고 26.Q2는 자동으로 아카이브 상태로 전환된다

**AC-10 (F-12 멤버 추가)**
- Given 멤버 관리 화면에 있는 상태에서
- When 새 멤버 이름을 추가하면
- Then 즉시 모든 KR의 Owner 선택 드롭다운에서 해당 멤버를 선택할 수 있다

**AC-11 (F-14, F-15 회고 항상 노출)**
- Given 분기 진행 중인 어느 시점에서든
- When 사용자가 페이지 하단으로 스크롤하면
- Then 점수표와 KPT 회고 섹션이 항상 보이며, 빈 칸은 placeholder로 안내된다

**AC-12 (F-16 PDF 다운로드)**
- Given 대시보드에 데이터가 입력된 상태에서
- When 사용자가 "PDF 다운로드" 버튼을 클릭하면
- Then 현재 화면 전체(헤더 + Hero 카드 + KR 카드 + 점수표 + KPT)가 PDF로 저장되고, Hero 카드와 색상 시스템이 인쇄에서도 식별 가능하게 유지된다

**AC-13 (F-17 영속화)**
- Given 편집자가 KR을 갱신해 저장한 상태에서
- When 다른 사용자가 다른 브라우저로 같은 URL에 접속하면
- Then 마지막 저장된 값이 동일하게 표시된다

**AC-14 (F-18 삭제 다이얼로그)**
- Given KR 카드에 삭제 버튼이 보이는 상태에서
- When 사용자가 삭제 버튼을 클릭하면
- Then "정말 삭제하시겠습니까?" 확인 다이얼로그가 뜨고, 확인 시에만 삭제된다

**AC-15 (수동 진척도)**
- Given KR 편집 화면에서
- When 사용자가 진척도 필드에 "65" 같은 숫자를 직접 입력하면
- Then 그 값이 그대로 저장되고 진척도 바에 65%로 반영된다 (자동 계산 없음)

## 12. 가정 및 확인 필요 사항

본문에서 [가정] 표기된 항목과 구현 시 자유도가 있는 항목을 정리.

| # | 항목 | 가정 내용 |
|---|---|---|
| A1 | PDF 생성 방식 | 브라우저 인쇄 API(`window.print()`) + 인쇄용 CSS를 1차 구현. 부족하면 html2canvas + jsPDF로 고도화 |
| A2 | 차트·게이지 렌더링 | 가벼운 HTML/CSS·SVG로 자체 구현. 외부 차트 라이브러리는 사용하지 않음 (번업·도넛 제거로 라이브러리 필요성 사라짐) |
| A3 | 한글 폰트 | Pretendard 또는 Noto Sans KR (PDF 출력 시 한글 깨짐 방지) |
| A4 | PDF 페이지 크기 | A4 세로 (보고 표준) |
| A5 | 인쇄용 색상 | 화면 색을 인쇄 친화 채도로 자동 조정, 흑백 인쇄 시에도 명도 차로 상태 구분 가능 |
| A6 | Vercel KV 접근 | Next.js API Route 경유, 클라이언트에 KV 키 미노출 |
| A7 | 분기 종료 후 상태 전환 | 새 분기 생성 시점에만 이전 분기를 archived로 전환. 종료일 도달만으로 자동 전환하지 않음 |
| A8 | KR/Objective 신규 추가 UI | 인라인 "+ 추가" 버튼 패턴 (각 Objective 하단에 KR 추가, 분기 본문 하단에 Objective 추가) |
| A9 | 진척도 입력 단위 | % 단위 정수 입력 (예: 65). 내부 저장은 0.65로 환산 |
