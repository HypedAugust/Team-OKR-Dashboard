# 사업전략팀 OKR 대시보드 IA (Information Architecture)

- **버전**: v1.0
- **작성일**: 2026-05-22
- **연계 문서**: [PRD.md](PRD.md)

---

## 0. 한 줄 요약

단일 페이지 구조(SPA-like)의 OKR 대시보드. 상단에서 하단으로 **요약 → 상세 → 회고** 순으로 정보가 흐르며, 분기 전환·멤버 관리·삭제 확인은 모달로 분리한다.

---

## 1. 사이트맵 (Sitemap)

대시보드는 단일 페이지 구조이며, URL 파라미터로 분기를 전환한다.

```
/                          → 메인 대시보드 (현재 활성 분기)
/?q={quarter_id}           → 특정 분기 조회 (예: /?q=26Q2)
```

> 별도 라우트는 사용하지 않는다. 모든 부가 액션(분기 생성·멤버 관리·삭제 확인)은 모달로 처리한다.

---

## 2. 페이지 구조 (Page Anatomy)

상단부터 하단까지 한 흐름으로 스크롤된다. 섹션 순서는 **정보 중요도 = 시각적 위계** 원칙에 따른다.

```
┌─────────────────────────────────────────────────────────────────────┐
│ [A] Global Header                                                    │
│  ├─ Logo / Title                                                     │
│  ├─ Quarter Selector ▼                                               │
│  └─ Action Buttons: [+ 새 분기] [👥 멤버 관리] [📄 PDF 다운로드]      │
├─────────────────────────────────────────────────────────────────────┤
│ [B] Quarter Meta Bar                                                 │
│  └─ 26.Q2 | 2026.04.01 ~ 06.30 | 경과 57% | 남은 6주 | 갱신 3/5    │
├─────────────────────────────────────────────────────────────────────┤
│ [C] Hero Summary (4 Cards)                                           │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                                │
│  │ 42%  │ │ 3/5  │ │  中  │ │ 57%  │                                │
│  │진척도 │ │ 달성 │ │신뢰도│ │경과율│                                │
│  └──────┘ └──────┘ └──────┘ └──────┘                                │
├─────────────────────────────────────────────────────────────────────┤
│ [D] OKR Tree                                                         │
│                                                                      │
│  [D-1] Objective Section (반복)                                      │
│   ├─ Objective Header: O1. 신규 수주 확보  ─ 평균 25% 🟡             │
│   └─ KR Card Grid                                                    │
│       ├─ [D-2] KR Card (반복)                                        │
│       │   ├─ Header: [Aspire] @ben @ria @que ─ 갱신 5/15            │
│       │   ├─ Body: 목표 / 현재값 / 진척도 바 / 신뢰도 신호등         │
│       │   └─ Status Badge: 🔴 위험 / ✓ 달성 완료 / (없음)            │
│       └─ [+ KR 추가] 버튼                                            │
│                                                                      │
│  [+ Objective 추가] 버튼                                             │
├─────────────────────────────────────────────────────────────────────┤
│ [E] 점수표 섹션 (항상 노출)                                          │
│  └─ KR별 행: 결과 / Commit·Aspire 평가                               │
├─────────────────────────────────────────────────────────────────────┤
│ [F] KPT 회고 섹션 (항상 노출)                                        │
│  └─ 멤버별 카드: Keep / Problem / Try                                │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. 영역별 정보 구성

### [A] Global Header

| 요소 | 목적 | 비고 |
|---|---|---|
| 로고/타이틀 | 페이지 식별 | "사업전략팀 OKR" 같은 고정 텍스트 |
| Quarter Selector ▼ | 분기 전환 | 활성 분기 + 아카이브 분기 모두 선택 가능 |
| + 새 분기 | 신규 분기 생성 | 클릭 → 모달 [M-1] |
| 👥 멤버 관리 | 멤버 추가·삭제 | 클릭 → 모달 [M-2] |
| 📄 PDF 다운로드 | 현재 화면 PDF 출력 | 클릭 시 즉시 다운로드 트리거 |

### [B] Quarter Meta Bar

| 요소 | 표시 형식 | 데이터 출처 |
|---|---|---|
| 분기명 | "26.Q2" | `quarter.name` |
| 기간 | "2026.04.01 ~ 06.30" | `quarter.start_date`, `end_date` |
| 경과율 | "경과 57%" | 자동 계산 (PRD §9-3) |
| 남은 주차 | "남은 6주" | 자동 계산 |
| 갱신율 | "갱신 3/5" | 자동 계산 (이번 주 월요일 0시 기준) |

### [C] Hero Summary

가장 큰 시각 요소. 4개 카드 가로 배열.

| 카드 | 큰 숫자 | 색상 기준 |
|---|---|---|
| 전체 평균 진척도 | "42%" | <30% 🔴 / 30~60% 🟡 / ≥60% 🟢 |
| 달성 KR 수 | "3/5" | 50% 미만 🔴 / 50~80% 🟡 / 80% 이상 🟢 [가정] |
| 평균 신뢰도 | "中" (또는 高·低) | 高 🟢 / 中 🟡 / 低 🔴 |
| 분기 경과율 | "57%" | 회색 (참고용, 색상 없음) |

### [D] OKR Tree

#### [D-1] Objective Section

| 요소 | 표시 |
|---|---|
| 번호·제목 | "O1. 신규 수주를 확보한다" |
| 평균 진척도 | "평균 25%" + 색상 배지 |
| 인라인 편집 | 제목 클릭 → 편집 모드 |
| 삭제 | 우상단 ✕ 버튼 → 확인 다이얼로그 [M-3] |
| + KR 추가 | 섹션 하단 점선 카드 |

#### [D-2] KR Card

세로 카드 형태. 좌측 색상 띠가 상태를 즉시 신호.

```
┌─────────────────────────────────────────┐
│ [🟢 좌측 띠]                            │
│  [Aspire]              👤 @ben @ria @que │  ← 헤더
│  ─────────────────────────────────────  │
│  목표:    응모·제안서 10건              │  ← 본문
│  현재값:  3건 (호반, IBK-다날, SBVA)    │
│  진척도:  ▓▓▓░░░░░░░ 30%               │
│  신뢰도:  🟡 中                         │
│  갱신:    2026-05-15                    │
│  ─────────────────────────────────────  │
│  ✓ 달성 완료  /  🔴 위험  /  (없음)     │  ← 상태 배지
└─────────────────────────────────────────┘
```

| 영역 | 요소 | 비고 |
|---|---|---|
| 좌측 띠 | 4px 두께 색상 띠 | 초록(달성) / 빨강(위험) / 회색(일반) |
| 헤더 | 유형 배지 + Owner 칩들 + 우측 삭제 ✕ | 유형 배지 색: Commit 파랑·Aspire 보라 [가정] |
| 본문 | 목표 / 현재값(숫자+상세) / 진척도 바 / 신뢰도 신호등 / 갱신일 | 클릭 → 인라인 편집 |
| 상태 배지 | "✓ 달성 완료" 또는 "🔴 위험" 또는 표시 안 함 | 우선순위: 달성 > 위험 |

### [E] 점수표 섹션

분기 진행 중에도 항상 노출. 빈 칸은 placeholder로 안내.

| 컬럼 | 내용 |
|---|---|
| KR | "O1-KR1" (자동 표시) |
| 유형 | Commit / Aspire (자동 표시) |
| 목표 | 자동 표시 |
| 결과 | 자유 텍스트 입력 |
| Commit/Aspire 평가 | 자유 텍스트 입력 |

### [F] KPT 회고 섹션

멤버 명단 기반 카드 그리드. 멤버 추가 시 카드 자동 생성.

```
┌──────────────────────────┐
│ @que                     │
│ ─────────────────────    │
│ Keep:     [____________] │
│ Problem:  [____________] │
│ Try:      [____________] │
└──────────────────────────┘
```

## 4. 모달 인벤토리 (Modal Map)

| ID | 트리거 | 내용 | 확인 동작 |

## 6. 상태 변형 (State Variants)

### 6-1. 분기 상태별

| 분기 상태 | 편집 가능 영역 | 비고 |
|---|---|---|
| **활성 (active)** | 모든 영역 | 헤더 액션 버튼 활성 |
| **아카이브 (archived)** | 없음 (전체 읽기 전용) | 인라인 편집·삭제·추가 버튼 비활성 / Hero·KR 카드 등 표시는 동일 |
| **데이터 없음** | 페이지 전체가 Empty State | "+ 새 분기 만들기" 단일 버튼만 노출 |

### 6-2. KR 카드 상태별

| 상태 조건 | 시각 표현 |
|---|---|
| 달성 완료 (Commit ≥1.0 OR Aspire ≥0.6) | 좌측 초록 띠 + ✓ + "달성 완료" 라벨 |
| 위험 (진척도 <30% OR 신뢰도 低) AND NOT 달성 | 좌측 빨간 띠 + 🔴 "위험" 배지 |
| 일반 | 좌측 회색 띠, 상태 배지 없음 |
| 미시작 (진척도 0% AND 갱신 없음) | 진척도 바 회색 |

### 6-3. 편집 모드

| 트리거 | 변경 |
|---|---|
| KR 카드 클릭 | 본문이 입력 폼으로 전환, 하단에 [저장] [취소] 버튼 노출 |
| Objective 제목 클릭 | 제목이 input으로 전환, Enter 또는 blur로 저장 |
| 점수표 셀 클릭 | 셀이 textarea로 전환 |
| KPT 칸 클릭 | 칸이 textarea로 전환 |

## 7. 정보 위계 (Information Hierarchy)

> 페이지를 처음 본 사람이 어떤 정보를 어떤 순서로 받게 되는지.

```
1순위 (즉시 인지, 0~3초)
  └─ [C] Hero Summary 4종 큰 숫자 + 색상
  └─ [B] 분기명·경과율·갱신율

2순위 (스캔, 3~10초)
  └─ [D-1] Objective별 평균 진척도 + 색상
  └─ [D-2] KR 카드의 좌측 색상 띠 / 상태 배지

3순위 (탐독, 10초~)
  └─ [D-2] KR 본문 (목표, 현재값, 진척도, 신뢰도, 갱신일)
  └─ [E] 점수표
  └─ [F] KPT 회고

4순위 (액션 트리거, 필요 시)
  └─ [A] Global Header 액션 버튼
  └─ 삭제 ✕ 버튼 (각 카드)
  └─ 인라인 편집 진입
```

## 8. 네비게이션 & 액션 흐름

### 8-1. 페이지 내 액션 → 결과 매핑

| 사용자 액션 | 트리거 위치 | 결과 |
|---|---|---|
| 분기 전환 | [A] Quarter Selector ▼ | URL 파라미터 변경, 같은 페이지에서 데이터만 교체 |
| 새 분기 생성 | [A] + 새 분기 | 모달 [M-1] 오픈 → 생성 → 페이지가 새 분기로 자동 전환 |
| 멤버 추가·삭제 | [A] 👥 멤버 관리 | 모달 [M-2] 오픈 → 변경사항 즉시 반영 |
| PDF 다운로드 | [A] 📄 PDF 다운로드 | 브라우저 인쇄 또는 PDF 저장 다이얼로그 |
| KR 갱신 | [D-2] 카드 클릭 | 인라인 편집 → 저장 → Hero 카드 즉시 갱신 |
| Objective/KR 추가 | [D] 하단 + 버튼 | 새 빈 카드 즉시 생성, 인라인 편집 진입 |
| Objective/KR/멤버 삭제 | 각 ✕ 버튼 | 확인 다이얼로그 [M-3] → 즉시 삭제 |
| 점수표/KPT 작성 | [E], [F] 칸 클릭 | 인라인 편집 |

### 8-2. 사용자 시나리오별 흐름

**시나리오 1: 상사가 분기 현황 확인**
```
URL 접속
  → [B] 분기명 확인
  → [C] Hero 4종 카드로 전체 상태 파악
  → [D-1] Objective별 색상으로 위험 영역 식별
  → (필요 시) [D-2] KR 본문 탐독
  → (보고용) [A] 📄 PDF 다운로드
```

**시나리오 2: 매주 금요일 갱신**
```
URL 접속
  → [B] 갱신율 "1/5" 확인 (본인 미갱신)
  → [D-2] 본인 담당 KR 카드 클릭
  → 현재값·진척도(%)·신뢰도(高/中/低) 입력
  → [저장]
  → Hero·갱신율 즉시 반영
```

**시나리오 3: 분기 종료 회고**
```
URL 접속 (예: 6/22 이후)
  → 스크롤 하단
  → [E] 점수표 KR별 결과·평가 입력
  → [F] KPT 본인 카드에 Keep/Problem/Try 작성
  → [A] 📄 PDF 다운로드 → 상사 보고
```

**시나리오 4: 새 분기 시작**
```
URL 접속
  → [A] + 새 분기 클릭
  → 모달 [M-1]: 분기명, 시작일, 종료일 입력
  → [생성]
  → 페이지가 새 분기(빈 폼)로 자동 전환, 이전 분기는 아카이브
  → [D] + Objective 추가 → 새 OKR 입력 시작
```

**시나리오 5: 과거 분기 조회**
```
URL 접속
  → [A] Quarter Selector ▼ → 26.Q1 선택
  → 모든 입력 UI 비활성화, 데이터만 표시
  → (보고 재활용 시) [A] 📄 PDF 다운로드
```

## 9. PDF 출력 구조

PDF는 화면 구조를 그대로 따른다. 단, 페이지 흐름을 자연스럽게 만들기 위해 다음 규칙 적용.

| 출력 순서 | 영역 |
|---|---|
| 1 | [B] Quarter Meta Bar |
| 2 | [C] Hero Summary 4종 |
| 3 | [D] OKR Tree (Objective 단위로 페이지 끊김 회피) |
| 4 | [E] 점수표 |
| 5 | [F] KPT 회고 |

> PDF에서는 [A] Global Header(액션 버튼)와 모든 편집 UI(삭제 ✕, + 추가, 인라인 편집 진입)는 숨긴다.

## 10. 엣지 케이스 화면

| 상황 | 화면 |
|---|---|
| 분기 0개 | Hero·트리·점수표·KPT 모두 숨김. 중앙에 "[+ 새 분기 만들기]" 단일 CTA |
| Objective 0개 (분기는 있음) | Hero 카드만 표시, 그 아래 "[+ Objective 추가]" 단일 CTA |
| KR 0개 (Objective는 있음) | Objective 헤더 + "[+ KR 추가]" 점선 카드만 표시 |
| 멤버 0명 | KPT 섹션에 "멤버를 먼저 추가해주세요" 안내 + 멤버 관리 모달 진입 링크 |
| 분기 종료 후 점수표 미작성 | 점수표 행마다 placeholder "분기 종료 후 작성하세요" |
| 진척도 0%이고 갱신 이력 없음 | 진척도 바 회색, 갱신일 "-" 표시 |
| 저장 실패 | 토스트 "저장 실패. 다시 시도하세요" + 폼 입력값 유지 |
| 과거 분기 진입 | 헤더에 "읽기 전용" 배지, 모든 편집 UI 비활성화 |
